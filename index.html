useRef

useRef is a really underestimated hook that provides you a way to create a mutable object that persists between renders. That way you can keep values around to use for internal purposes on your components and hooks. Its similar to useState in the fact that it holds some sort of state for the component. 


const ref = useRef("üêç");

ref.current === üêç" // true

This will create a object that has a current property on it, and thats where you can store what ever values you want. The API is pretty simple, the value you pass into the hook will be set as the internal value, note this value passed in will only be set in the first render.

Mutating the useRef object will not trigger a rerender and thats its gratest feature. The example below shows the implementation of a counter using useRef. Note that this will not work like useState does.


function CounterRef() {
  const countRef = React.useRef(0);
  const rerender = useRerender();
  return (
    <div>
      <h1>{countRef.current}</h1>
      <button onClick={() => countRef.current--}>-</button>
      <button onClick={() => countRef.current++}>+</button>
      <button onClick={rerender}>Rerender</button>
    </div>
  );
}

Accessing DOM

The most common use case you will see for useRef is for accessing te undelying DOM element that is behind the JSX we write. That way we can interact directly with the DOM element if needed.

Note that you should only manipulate the element inside the ref after render. Since the DOM element has not been created yet, before the render occurs.


export function AccessDOM() {
  const domRef = React.useRef(null);
  const [hide, setHide] = React.useState(false);
  const [el, setEl] = React.useState(domRef.current);

  const toggle = () => setHide((h) => !h);

  React.useLayoutEffect(() => {
    setEl(domRef.current &&  domRef.current.nodeName )
  }, [hide]);

  return (
    <div>
      {!hide && <div ref={domRef}>Hello üëã</div>}
      <span >{el ? `Node name is a ${el}` : `Node is ${el}`}</span>
      <Button onClick={toggle}>Toggle Hide</Button>
    </div>
  );
}

useRef and dependency array

You can use the useRef hook as a bail out for passing something into the dependency array that you know will be different every render. You want to have the latest value of the unstable value, but not invalidate the dependency array just because of it.

In this case below you will have this useEffect run on every render just because the inline callback is a new one on every render. If you only omit from the dependency array and break the rules of hooks you might end up with a stale function inside the useEffect, that point to stale variables from previous renders.


const useHook(aInlineCallback) {
  // more stuff here
  React.useEffect(() => {
    aInlineCallback(other, stable, deps)
  }, [other, stable, deps, aInlineCallback])
  // more stuff here
}
// hook usage
useHook(() => (...))

How to bail out of dependency array.

In this case below you will have the useEffect run only when the stable values on the dependency array changes, but you still get the benefits of having the most up to date funtion.

NOTE: if you want you effect to run when the function changes you should tell the user to pass the function wrapped in useCallback.


const useHook(aInlineCallback) {
  const callbackRef = React.useRef(aInlineCallback);

  React.useLayoutEffect(() => {
    callbackRef.current = aInlineCallback
  });
  // more stuff here
  React.useEffect(() => {
    callbackRef.current(other, stable, deps)
  }, [other, stable, deps]);
  // more stuff here
}
// hook usage
useHook(() => (...));

useImperativeHandle ‚Ü©Ô∏è

First let look at this quote from the React docs:

useImperativeHandle customizes the instance value that is exposed to parent components when using ref. As always, imperative code using refs should be avoided in most cases. useImperativeHandle should be used with forwardRef

So in essence useImperativeHandle gives us a way to break the trade mark of React which is one way data flow. It gives you the power to access internal values from the child on the parent component

As the first paremeter you get the forwarded ref, the second paremeter is a calback with the value you want to inject in the ref, and similar to the useEffect/useLayoutEffect hooks, it has a third paremeter that is a dependency array so you can let the hook know when it should update the injected values. The undelying implementation of useImperativeHandle uses useLayoutEffect, so it run after DOM updates but before paint.


React.useImperativeHandle(ref, () => "üöÄ",[]);

Quick example

Here you can see how we are able to make the parent component control the child input focus. This API is mostly used for modals, where the state of the modal is held in the child, but you still want the parent to control it.


const Input = React.forwardRef((props, ref) => {
  const inputRef = React.useRef(null)

  React.useImperativeHandle(ref, () => ({ focus: () => inputRef.current.focus() }), []);

  return <input ref={inputRef} {...props} />;
});

export function InputFocus() {
  const inputRef = React.useRef(null);

  return (
    <div>
      <Input ref={inputRef} placeholder="useImperativeHandle" />
      <Button onClick={() => inputRef.current.focus()}>Focus</Button>
    </div>
  );
}

export function Confetti() {
  const [show, setShow] = React.useState(false);
  const { width, height } = useWindowSize();
  const timerRef = React.useRef(0);

  const handleClick = () => {
    setShow(true);
    clearTimeout(timerRef.current);
    timerRef.current = setTimeout(() => {
      setShow(false);
    }, 10000);
  };

  return (
    <div>
      <Button onClick={handleClick}>Click here!</Button>
      {show ? <ReactConfetti width={width} height={height} /> : null}
    </div>
  );
}
